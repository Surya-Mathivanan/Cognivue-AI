================================================================================
                         COGNIVUE AI — PROJECT DETAILS
================================================================================

Project Name  : Cognivue AI
Type          : Full-Stack Web Application (AI-Powered Interview Coach)
Frontend      : React (JSX) + Vite
Backend       : Django (Python)
AI Engine     : Google Gemini API (gemini-3-flash-preview)

Deployment    : Frontend → Vercel | Backend → Render | DB → PostgreSQL

================================================================================
                              PROJECT CORE CONCEPT
================================================================================

Cognivue AI is an AI-powered mock interview platform with two user types:

  1. STUDENTS / CANDIDATES
     - Sign in via Google OAuth
     - Get a unique Candidate UID (format: 22BAD + 5 digits, e.g. 22BAD12345)
     - Practice mock interviews in two modes:
         a) Resume-Based  — Upload a PDF resume; AI extracts skills & projects
                            and generates hyper-personalized questions
         b) Role-Based    — Pick a job role (e.g. Frontend Developer) and
                            experience level; AI generates domain-specific Qs
     - Answer questions via text or voice (Web Speech API)
     - Receive AI-scored feedback: overall score, category breakdown,
       strengths, improvements, and a detailed written analysis
     - View full session history with pagination and per-session drill-down
     - Manage a profile (UID, college, degree, branch, skills, contact info)

  2. HR PROFESSIONALS
     - Register with email + OTP verification (via EmailJS)
     - Log in via email/password
     - Search candidates by UID or by Job Role
     - View a candidate's full interview Q&A, scores, and AI feedback
     - Used for talent screening — candidates share their UID with recruiters

================================================================================
                              BACKEND ARCHITECTURE
================================================================================

The backend is a Django REST application split into 4 apps:

--------------------------------------------------------------------------------
App: accounts  (backend/accounts/)
--------------------------------------------------------------------------------
Purpose : User accounts, Google OAuth, JWT authentication, session management.

Models:
  - User (extends AbstractUser)
      Fields: email (unique, USERNAME_FIELD), username, avatar_url,
              google_id, uid (10 chars, auto-generated: "22BAD" + 5 digits),
              college_name, degree, branch, address, phone_number,
              skills, location
      Auto-generates a unique UID on first save with up to 10 retry attempts.

Key Endpoints (backend/accounts/urls.py + views.py):
  - GET  /auth/google/             — Redirects to Google OAuth consent page
  - GET  /auth/google/callback/    — Handles OAuth callback; issues JWT;
                                     redirects to frontend with ?token=<jwt>
  - GET  /api/user-info/           — Returns authenticated user's profile JSON
  - POST /api/logout/              — Clears server-side session (JWT is stateless)
  - POST /api/update-profile/      — Updates editable profile fields

JWT Utilities (backend/accounts/jwt_utils.py):
  - generate_jwt(user)    — Signs a JWT with user ID + 7-day expiry
  - verify_jwt(token)     — Decodes and validates the JWT
  - Uses SECRET_KEY from Django settings as HMAC-SHA256 signing key

--------------------------------------------------------------------------------
App: interviews  (backend/interviews/)
--------------------------------------------------------------------------------
Purpose : Interview session lifecycle, question generation, answer storage,
          AI-powered feedback scoring.

Models:
  - InterviewSession
      Fields: user (FK), mode ('resume'|'role'), difficulty
              ('beginner'|'intermediate'|'advanced'), role,
              resume_filename, technical_skills (JSON), soft_skills (JSON),
              projects (JSON), experience_level, resume_summary,
              questions (JSON), answers (JSON), feedback (JSON),
              status ('active'|'completed'), created_at, completed_at
      Properties: overall_score (from feedback JSON), duration_minutes

Key Endpoints (backend/interviews/urls.py + views.py):
  - POST /api/upload-resume/       — Accepts PDF; runs ResumeAnalyzer;
                                     returns extracted keywords
  - POST /api/generate-questions/  — Accepts interviewData; runs
                                     QuestionGenerator; creates session;
                                     returns questions + session_id
  - POST /api/submit-answer/       — Saves one answer to session
  - POST /api/complete-interview/  — Triggers Gemini AI feedback scoring;
                                     marks session completed; returns feedback
  - GET  /api/session-history/     — Returns all completed sessions for user
  - GET  /api/analytics/           — Returns aggregated stats (totals, avg score,
                                     best score, by difficulty, resume/role counts)
  - GET  /api/session/<id>/        — Returns full session detail (questions,
                                     answers, feedback) for a specific session

--------------------------------------------------------------------------------
App: hr  (backend/hr/)
--------------------------------------------------------------------------------
Purpose : HR professional accounts, OTP authentication, candidate search API.

Models (backend/hr/models.py):
  - HRUser: name, email, company, password_hash, is_verified,
            otp_code, otp_created_at
  - (Reads from accounts.User for candidate data)

Key Endpoints (backend/hr/urls.py + views.py):
  - POST /hr/register/             — Creates HRUser; generates OTP; returns
                                     OTP code for EmailJS to send
  - POST /hr/verify-otp/           — Verifies 6-digit OTP; marks as verified;
                                     sets session cookie for HR
  - POST /hr/login/                — Authenticates HR by email+password;
                                     sets session cookie
  - POST /hr/logout/               — Clears HR session cookie
  - POST /hr/resend-otp/           — Generates new OTP for email
  - GET  /hr/me/                   — Returns logged-in HR user info
  - GET  /hr/roles/                — Returns list of distinct roles from all
                                     completed interview sessions
  - GET  /hr/search/uid/?uid=X     — Returns candidate profile + all their
                                     sessions matching the UID
  - GET  /hr/search/role/?role=X   — Returns all candidates with completed
                                     sessions for the given role
  - GET  /hr/session/<id>/         — Returns full session detail + candidate
                                     info for HR inspection

Email Service (backend/hr/email_service.py):
  - OTP delivery handled client-side via EmailJS SDK in HRLoginScreen.jsx
  - The backend generates and stores the OTP; EmailJS sends the email

--------------------------------------------------------------------------------
App: core  (backend/core/)
--------------------------------------------------------------------------------
Purpose : AI integrations — resume parsing and question generation.

Files:
  gemini.py
  - Initializes the Google genai.Client using GEMINI_API_KEY from settings
  - Provides a singleton client instance used by both modules

  resume_analyzer.py  (ResumeAnalyzer class)
  - Parses PDF resumes using PyPDF2 or pdfplumber
  - Sends raw text to Gemini AI with a structured extraction prompt
  - Returns JSON with:
      technical_skills: [{name, proficiency, category}]
      soft_skills: [{skill, evidence}]
      projects: [{title, description, technologies, role, impact}]
      experience_level: beginner | intermediate | senior
      resume_summary: one-paragraph summary

  question_generator.py  (QuestionGenerator class)
  - RESUME-BASED MODE:
      generate_resume_based_questions(technical_skills, soft_skills, projects, difficulty)
      → _generate_technical_questions() — Prompts Gemini for EXACTLY 5 Qs
                                          about listed tech skills, difficulty-adjusted
      → _generate_hr_questions()        — Prompts Gemini for EXACTLY 4 behavioral Qs
                                          including ≥2 STAR method questions
      → _generate_project_questions()   — Prompts Gemini for EXACTLY 3 project Qs
                                          probing depth of involvement
      All use response_mime_type="application/json" and retry up to 2 times.
      Raises Exception if LLM fails after retries (no silent fallbacks).

  - ROLE-BASED MODE:
      generate_role_based_questions(role, difficulty)
      → Prompts Gemini for:
          3 HR/behavioral questions
          4 technical questions
          3 situational/cultural fit questions
      Uses regex JSON extraction; retries up to 3 times with exponential
      backoff; returns error dict if all attempts fail.

  - Model used: "gemini-3-flash-preview" with temperature=0.7

================================================================================
                          AUTHENTICATION FLOWS SUMMARY
================================================================================

STUDENT LOGIN FLOW:
  1. User clicks "Continue with Google" → redirected to /auth/google/
  2. Google OAuth consent → /auth/google/callback/ on backend
  3. Backend creates/updates User record, generates JWT
  4. Redirects to frontend /?token=<jwt>
  5. App.jsx reads ?token=, saves to localStorage, cleans URL
  6. All subsequent API calls send: Authorization: Bearer <token>

HR LOGIN FLOW (existing account):
  1. Enter email + password → POST /hr/login/
  2. Backend validates, sets session cookie
  3. Navigate to /hr-admin

HR REGISTER FLOW (new account):
  1. Fill Name/Email/Company/Password → POST /hr/register/
  2. Backend creates HRUser, generates OTP, returns otp_code + email
  3. Frontend sends OTP email via EmailJS SDK
  4. User enters 6-digit OTP → POST /hr/verify-otp/
  5. Backend verifies OTP, marks HR as verified, sets session cookie
  6. Navigate to /hr-admin

================================================================================
                         INTERVIEW SESSION FLOW SUMMARY
================================================================================

RESUME-BASED:
  ResumeUpload (select PDF + difficulty)
    → POST /api/upload-resume/ (ResumeAnalyzer extracts skills/projects)
    → InterviewSession (questions generated via POST /api/generate-questions/)
    → User answers each question (text or voice recording)
    → Each answer saved via POST /api/submit-answer/
    → On last answer: POST /api/complete-interview/ (Gemini scores all answers)
    → FeedbackDashboard (overall score, category scores, strengths, improvements)
    → Saved in InterviewSession DB record

ROLE-BASED:
  RoleSelection (choose role + difficulty)
    → InterviewSession (questions generated via POST /api/generate-questions/)
    → [Same as above from step 3]
